# Кейс-задача 2.4 — Анализ рынка веб-информационных систем

## 1. Цель анализа

Оценить существующие на рынке программные решения для информационных систем на веб-архитектуре и определить, как их возможности могут быть использованы в компании (на примере туристического агентства).

## 2. Обзор рынка: типы решений

| Категория | Примеры | Назначение |
|-----------|---------|------------|
| CRM для туризма | TravelLine, Amadeus Selling Platform, Sabre | Управление бронированиями, клиентами, тарифами |
| ERP / учёт | 1С:Тourism, SAP for Travel | Финансы, кадры, отчётность |
| Конструкторы сайтов | WordPress + WooCommerce, Tilda | Быстрый запуск витрины туров |
| Low-code платформы | Microsoft Power Apps, OutSystems | Внутренние ИС без глубокой разработки |
| Кастомная разработка | Delphi + IIS, ASP.NET, PHP/Laravel | Полный контроль логики и интеграций |

## 3. Сравнительная оценка возможностей

### 3.1 TravelLine / Amadeus (отраслевые платформы)

**Возможности:**
- Интеграция с авиакомпаниями и отелями
- Онлайн-бронирование и оплата
- Личный кабинет туриста
- Аналитика загрузки и выручки

**Применение в компании:**
- Подходит для среднего и крупного агентства с большим потоком бронирований
- Снижает время обработки заявки
- Требует абонентской платы и обучения персонала

**Ограничения:** высокая стоимость, зависимость от поставщика, сложная кастомизация.

### 3.2 1С:Тourism

**Возможности:**
- Учёт договоров, платежей, взаиморасчётов
- Интеграция с бухгалтерией
- Отчёты для руководства

**Применение в компании:**
- Оптимально для бэк-офиса: бухгалтерия, договоры, зарплата менеджеров
- Не заменяет полноценный публичный сайт для клиентов

### 3.3 WordPress + плагины бронирования

**Возможности:**
- Быстрый запуск сайта-каталога
- SEO, блог, формы заявок
- Низкий порог входа

**Применение в компании:**
- Стартап или малое агентство
- Маркетинг и привлечение клиентов

**Ограничения:** безопасность зависит от плагинов, сложная бизнес-логика реализуется с трудом.

### 3.4 Кастомное решение (Delphi 10.2 + IIS + MS SQL Server)

**Возможности:**
- Полный контроль интерфейса и логики
- Интеграция с внутренней БД компании
- Развёртывание на собственном сервере Windows

**Применение в компании:**
- Внутренняя ИС менеджеров + публичная часть для клиентов
- Компания с IT-отделом или подрядчиком
- Требования к безопасности данных на своём сервере

## 4. Вывод: что выбрать компании

| Потребность компании | Рекомендуемое решение |
|---------------------|----------------------|
| Быстро показать туры в интернете | WordPress / Tilda |
| Автоматизировать бронирования и GDS | TravelLine / Amadeus |
| Бухгалтерия и договоры | 1С |
| Уникальные процессы, своя БД, Windows-инфраструктура | **Delphi + IIS + MS SQL Server** (наше задание) |

## 5. Обоснование выбора для учебного проекта

Для кейса выбрано **кастомное веб-приложение «Туристическое агентство»** на стеке Delphi 10.2 + IIS + MS SQL Server, потому что:

1. Демонстрирует полный цикл: анализ → проектирование БД → серверное приложение → публикация.
2. Позволяет реализовать роли (администратор, менеджер, гость).
3. Данные хранятся в корпоративной СУБД MS SQL Server.
4. IIS — стандартный веб-сервер Windows, используемый в enterprise-среде.

## 6. Функциональные требования к разрабатываемой системе

### Публичная часть (для клиентов)
- Просмотр каталога опубликованных туров
- Фильтр по направлению и категории
- Форма заявки на бронирование

### Административная часть (для менеджеров)
- Авторизация по логину и паролю
- CRUD туров (создание, редактирование, публикация)
- Просмотр и обработка заявок (подтверждение / отмена)

### Технические требования
- Сервер приложений: IIS 10+
- СУБД: MS SQL Server 2019+
- Клиент: любой современный браузер
- Кодировка: UTF-8 / Unicode

## 7. Архитектура решения

```
[Браузер клиента]
       |
       v
[ IIS — веб-сервер Windows ]
       |
       v
[ Delphi WebBroker / IntraWeb / DataSnap REST ]
       |
       v
[ MS SQL Server — база TourismWebApp ]
```

**Слои приложения:**
1. **Presentation** — HTML-страницы / формы IntraWeb
2. **Business Logic** — модули Delphi (авторизация, бронирование)
3. **Data Access** — FireDAC / ADO → хранимые процедуры и SQL
4. **Database** — таблицы Roles, Users, Destinations, TourCategories, Tours, Bookings

Подробная пошаговая инструкция по установке и разработке — в файле [TZ_STEP_BY_STEP.md](TZ_STEP_BY_STEP.md).
