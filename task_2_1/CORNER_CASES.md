# Граничные случаи — задача 2.1

Правило: первый индекс `max(A)` и первый индекс `min(A)` → элементы строго между ними → сумма `< 0`.

## Размер массива

| ID | Ситуация | Пример | Ответ |
|----|----------|--------|-------|
| CC-01 | Пустой массив | `[]` | `0` |
| CC-02 | Один элемент | `[5]` | `0` |
| CC-03 | Два элемента (индексы 0 и 1) | `[5, -2]` | `0` |

При `N = 2` max и min всегда соседи — между ними пусто.

## Положение max и min

| ID | Пример | Max | Min | Между | Ответ |
|----|--------|-----|-----|-------|-------|
| CC-10 | `[8, -1, -2, -3, 2]` | 8 @0 | -3 @3 | -1, -2 | **-3** |
| CC-11 | `[1, -4, 3, 9, 2]` | 9 @3 | -4 @1 | 3 | **0** |
| CC-12 | `[3, -1, -2, 7, -4, 1]` | 7 @3 | -4 @4 | — | **0** |
| CC-13 | `[5, -2, -3, -1]` | 5 @0 | -3 @2 | -2 | **-2** |
| CC-14 | `[-5, 2, 3, 10]` | 10 @3 | -5 @0 | 2, 3 | **0** |
| CC-15 | `[1, 2, 10, -1, -2, 3, 4]` | 10 @2 | -2 @4 | -1 | **-1** |

## Состав промежутка

| ID | Пример | Ответ | Причина |
|----|--------|-------|---------|
| CC-20 | `[20, -3, -5, -7, 10]` | **-8** | -3 + -5 |
| CC-21 | `[1, 5, 2, 9, 3]` | **0** | нет отрицательных |
| CC-22 | `[5, 0, 0, -3, 1]` | **0** | 0 не `< 0` |
| CC-23 | `[10, 0, -1, 5, -5, 2]` | **-1** | только -1 |
| CC-24 | `[-1, -5, -2, -3]` | **0** | max @0, min @1 — соседи |
| CC-25 | `[-1, -5, -2, -8]` | **-7** | -5 + -2 |

## Дубликаты

| ID | Пример | Правило | Ответ |
|----|--------|---------|-------|
| CC-30 | `[9, -1, -2, 1, 5, 9]` | первый max @0, min @2 | **-1** |
| CC-31 | `[5, -2, 3, -2, 1]` | первый min @1, max @0 — соседи | **0** |
| CC-32 | `[4, 4, 4]` | max и min @0 | **0** |

CC-30: если взять **последний** max @5, промежуток другой → **0**. Берём первое вхождение.

## Дроби

| ID | Пример | Ответ |
|----|--------|-------|
| CC-40 | `[5.0, -1.2, 2.0, -4.1]` | **-1.2** |

## Ошибки ввода (`input()`)

| ID | Ввод | Результат |
|----|------|-----------|
| EX-01 | N = `abc` | «Ошибка ввода» |
| EX-02 | N ≤ 0 | «Размер массива должен быть положительным» |
| EX-03 | нечисловые элементы | «Ошибка ввода» |
| EX-04 | элементов ≠ N | «Ожидалось N элементов…» |

Функция `sum_negative_between_extremes` на корректном списке исключений не бросает.

## Не обрабатывается

| ID | Случай |
|----|--------|
| LIM-01 | Пустой массив через API |
| LIM-02 | NaN, Infinity |
| LIM-03 | «Между» по значению, а не по индексу |

## Трактовка условия

| Вопрос | Решение |
|--------|---------|
| «Между» | по **индексам** |
| Несколько max/min | **первое** вхождение |
| max и min в сумму | **нет** |
| Ноль | не отрицательный |
